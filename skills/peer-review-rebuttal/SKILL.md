# Peer Review Rebuttal

Use this skill after the paper exists and the reviews arrive.

## Core Workflow

1. Classify comments:
   - misunderstanding,
   - missing evidence,
   - real weakness,
   - scope mismatch.
2. Draft the factual answer first, then the tone.
3. Propose only revisions you can actually deliver.
4. Align every response with the paper text and available evidence.
5. End with a revision plan by priority.

## Execution Rules

1. Never dodge a correct reviewer criticism.
2. Correct misunderstandings with citation to the paper text or experiment table.
3. If evidence is missing, say what can be added and what cannot be claimed.
4. Keep tone precise, calm, and non-combative.

## Output Contract

Return:

1. Comment classification.
2. Draft responses.
3. Required paper edits.
4. Extra experiment plan if needed.
5. Residual risk after rebuttal.
